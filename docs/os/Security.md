## Security

**Security(보안)** 는 사용자 인증을 통해 데이터와 코드의 무결성, 그리고 시스템의 물리적 자원을 보호하는 것이다. **Protection(보호)** 은 운영체제로부터 적절한 권한을 부여받은 프로세스만
메모리, CPU 등 시스템 자원에 접근할 수 있도록 제어하는 것이다. Security는 외부 위협으로부터 시스템을 지키는 것이고, Protection은 시스템 내부에서 프로세스와 자원 간의 접근을 제어한다는 점에서
차이가 있다.

### Computer Resources

컴퓨터 자원은 실수나 의도적인 행동에 의해 오용될 수 있기 때문에 이를 방지하고 탐지하기 위한 보안 메커니즘이 필요하다. 보안은 컴퓨터 자원을 권한 없는 접근, 악의적인 파괴나 변경, 의도하지 않은 데이터
불일치로부터 보호하는 것을 의미하며, 암호학은 이를 가능하게 하는 핵심 기술이다.

### Security Violations

Security Violation은 시스템 보안이 침해되는 상황을 의미한다. **Threat** 는 보안 침해가 발생할 가능성이고, **Attack** 은 보안을 깨기 위한 의도적인 시도다. 보안 침해의 유형으로는
기밀성(Confidentiality) 침해, 무결성(Integrity) 침해, 가용성(Availability) 침해, 서비스 도용, 서비스 거부 공격(DoS/DDoS)이 있다.

### Four Levels of Security

컴퓨터 보안은 네 가지 수준으로 나뉜다. **물리적 보안** 은 시스템에 대한 직접적인 접근을 막는 것이고, **네트워크 보안** 은 네트워크를 통한 공격을 방지하는 것이다. **운영체제 보안** 은 공격 표면을
줄여 침입을 방지하고, **애플리케이션 보안** 은 프로그램 내 보안 취약점을 관리하는 것이다.

### Security Holes: 보안 취약점

**Malware(악성 소프트웨어)** 는 컴퓨터 시스템을 악용하거나 손상시키기 위해 설계된 소프트웨어다.

> **트로이 목마(Trojan Horse)** : 유용한 프로그램으로 위장해 사용자가 스스로 설치하게 만든다. 설치되면 시스템 파괴나 정보 유출의 통로가 된다.
> **스파이웨어(Spyware)** : 몰래 설치되어 키로깅, 개인 정보 등을 수집해 공격자에게 전송한다.
> **랜섬웨어(Ransomware)** : 파일을 암호화해 사용 불가 상태로 만든 뒤 복구 대가로 금전을 요구한다.
> **백도어(Backdoor)** : 인증 없이 시스템에 접근할 수 있도록 몰래 만들어둔 통로다.
> **논리 폭탄(Logic Bomb)** : 특정 조건이 충족되는 순간 데이터 파괴 등의 악성 동작을 수행한다.

**Code Injection(코드 인젝션)** 은 입력값 검증이 불충분할 때 공격자가 악의적인 코드를 삽입하여 실행시키는 공격 방식이다. 악의적으로 만들어진 소프트웨어가 아니더라도 이 공격에 취약할 수 있다.

**바이러스(Virus)** 는 정상 프로그램 안에 숨어 자기 복제를 통해 다른 프로그램을 감염시키는 코드다. **웜(Worm)** 은 사람의 개입 없이 네트워크를 통해 스스로 복제하고 전파된다는 점에서 바이러스와
구별된다.

### Threats in the System and the Network

**Network Traffic 공격** 은 네트워크 트래픽을 이용한 공격 방식이다. **Sniffing** 은 공격자가 네트워크 트래픽을 수동적으로 도청하여 데이터를 가로채는 방식이고, **Spoofing** 은
통신 당사자 중 하나로 위장하거나 두 당사자 사이에 끼어들어 트랜잭션을 가로채고 변조하는 **중간자 공격(Man-in-the-Middle)** 방식이다.

**DoS(Denial of Service)** 는 정보 탈취나 자원 도용이 목적이 아니라, 시스템이나 서비스를 정상적으로 사용하지 못하도록 방해하는 공격이다. 예를 들어 수강신청 사이트에 1초에 수만 번의 요청을
보내 서버를 마비시켜 정상적인 사용자가 접근하지 못하게 만드는 것이다.

**Port Scanning** 은 그 자체가 공격은 아니지만 시스템의 취약점을 탐지하는 수단이다. 열려있는 포트를 파악하여 어떤 서비스가 동작 중인지 확인하고, 운영체제의 종류를 파악하는 **핑거프린팅(
Fingerprinting)** 의 일부로 활용된다. 실제 공격에 앞서 정보를 수집하는 단계에서 주로 사용되며, **방화벽(Firewall)** 을 통해 불필요한 포트를 차단하여 방어할 수 있다.

## Cryptography

암호학(Cryptography)은 네트워크 환경에서 메시지의 신뢰성을 보장하기 위한 기술이다. 네트워크에는 메시지를 보내는 송신자와 받는 수신자가 존재하며, 암호학은 네트워크 자체를 신뢰하지 않아도 안전하게 통신할
수 있도록 한다.

암호학은 키(Key) 라는 비밀값을 기반으로 동작한다. 키는 네트워크의 특정 컴퓨터에만 선택적으로 배포되며, 메시지를 암호화(Encrypt)하거나 복호화(Decrypt)하는 데 사용된다.

## Encryption (암호화)

암호화는 키를 가진 수신자만 메시지를 읽을 수 있도록 평문을 암호문으로 변환하는 기술이다. 암호화 시스템은 키(K), 메시지(M), 암호문(C)으로 구성되며, 암호화 함수(E)는 메시지를 암호문으로 변환하고 복호화
함수(D)는 암호문을 다시 원래 메시지로 복원한다.

- K (Keys) : 키의 집합
- M (Messages) : 평문 메시지의 집합(평서문)
- C (Ciphertexts) : 암호화된 메시지의 집합(암호문)
- E : K → (M → C) : 키를 사용해 평문을 암호문으로 변환하는 암호화 함수
- D : K → (C → M) : 키를 사용해 암호문을 평문으로 복원하는 복호화 함수

메시지(M)를 암호화 함수(E)와 키(K)를 사용해 암호문(C)으로 변환하여 송신하고, 수신 측에서는 복호화 함수(D)와 키(K)를 사용해 암호문(C)을 다시 원래 메시지(M)로 복원한다.

```text
송신 : M + K → E → C
수신 : C + K → D → M
```

암호문 c가 주어졌을 때, 키 k를 가진 컴퓨터만 `E_k(m) = c`를 만족하는 평문 m을 계산할 수 있다. 즉 키를 가진 컴퓨터는 암호문을 복호화할 수 있지만, 키를 갖지 않은 컴퓨터는 복호화가 불가능하다.
암호문은 일반적으로 네트워크에 노출되므로, 암호문만으로 키 k를 역추적하는 것이 현실적으로 불가능해야 한다는 점이 중요하다.

암호화와 복호화에 동일한 키를 사용하는 대칭 암호화(Symmetric Encryption) 방식과 암호화와 복호화에 서로 다른 키를 사용하는 비대칭 암호화(Asymmetric Encryption) 방식이 있다.

### Symmetric Encryption

대칭 암호화는 암호화와 복호화에 동일한 키를 사용하는 방식이다. 키 하나로 처리하므로 속도가 빠르다는 장점이 있지만, 송신자와 수신자가 반드시 동일한 키를 공유해야 한다는 것이 가장 큰 문제다. 키 교환은 두
당사자가 직접 하거나 **인증 기관(Certificate Authority)** 과 같은 신뢰할 수 있는 제3자를 통해 이루어진다. 키가 노출되면 모든 통신이 위험해지므로 키 관리가 중요하다.

대표적인 알고리즘으로는 **DES(Data Encryption Standard)** 와 **AES(Advanced Encryption Standard)** 가 있으며, 높은 수준의 보안이 요구되지 않는 경우 현재도
널리 사용된다.

> 1. DES (Data Encryption Standard): 데이터를 64비트 블록으로 나누어 암호화하는 블록 암호입니다.
> 2. AES (Advanced Encryption Standard): DES와 같은 블록 암호이지만, 블록 크기가 128비트로 더 큽니다.

그러나 키를 교환할 때 탈취당한다면 문제가 발생한다. 이를 해결하기 위해 등작한 것이 비대칭 암호화이다.

### Asymmetric Encryption

비대칭 암호화(Asymmetric Encryption)는 암호화와 복호화에 서로 다른 키를 사용하는 방식이다.
외부에 공개하는 공개 키(Public Key) 로 암호화하고, 본인만 보유하는 개인 키(Private Key) 로 복호화한다. 누구나 공개 키로 메시지를 암호화할 수 있지만, 복호화는 개인 키를 가진 수신자만 할 수
있다.

핵심은 공개 키와 개인 키가 수학적으로 긴밀하게 연결되어 있지만, 공개 키를 알고 있다고 해서 개인 키를 역추적할 수 없다는 점이다. 이는 일방향 함수(Trapdoor One-way Function) 를 기반으로
동작하기 때문이다. 대표적인 RSA 알고리즘 은 두 큰 소수를 곱하는 것은 쉽지만, 곱해진 결과만으로 원래 소수를 역추적하는 것은 현실적으로 불가능하다는 소인수분해의 어려움을 이용한다.

개인 키를 외부에 공유할 필요 없이 공개 키만 배포하면 되므로, 대칭 암호화의 키 교환 문제를 해결할 수 있다. 단, 계산량이 많아 대칭 암호화보다 처리 속도가 느리다는 단점이 있다.

### Authentication (인증)

암호화가 메시지를 받을 수 있는 수신자를 제한하는 기술이라면, 인증(Authentication) 은 메시지를 보낼 수 있는 송신자를 제한하는 기술이다. 메시지가 신뢰할 수 있는 송신자로부터 왔음을 확인하고, 전송
도중 변조되지 않았음을 검증하는 데 활용된다.

인증의 핵심은 원본 메시지를 역추적할 수 없도록 보호하는 것이다. 이를 위해 해싱(Hashing) 을 사용한다. 해싱은 원본 데이터(A)를 해시값(A')으로 변환하는 단방향 함수로, A'으로부터 A를 역추적하는 것이
현실적으로 불가능하다. 비밀번호 저장이 대표적인 예로, 원본 비밀번호 대신 해시값만 저장하여 원본을 보호한다.

인증 알고리즘은 키(K), 메시지(M), 인증자(A)로 구성된다. 해시 함수 `S : K → (M → A)`는 키를 사용해 메시지로부터 인증자를 생성하고, 검증 함수
`V : K → (M × A → {true, false})`는 메시지와 인증자의 쌍이 유효한지 확인한다. 키 k를 가진 컴퓨터만 `V_k(m, a) = true`를 만족하는 인증자를 생성할 수 있으며, 키가 없는
컴퓨터는 인증자를 생성할 수 없다.

인증 알고리즘은 세 가지 방식으로 구현된다.  
Hash Function(해시 함수)은 메시지로부터 고정된 크기의 메시지 다이제스트(Message Digest) 를 생성한다.  
MAC(Message Authentication Code)은 비밀 키를 사용하여 암호화된 체크섬을 생성하고 메시지 변조 여부를 검증한다.  
Digital Signature(디지털 서명)은 누구나 메시지의 진위를 검증할 수 있도록 서명을 생성하는 방식으로, 공개 키로부터 개인 키를 역추적하는 것이 불가능하다는 점을 이용한다. 비트코인의 블록체인이 대표적인
활용 사례다.

## Protection

보호(Protection)는 컴퓨터 시스템에서 정의된 자원에 대해 프로세스와 사용자의 접근을 제어하는 것이다. 공유 자원을 사용하거나 인터넷과 같은 보안이 취약한 통신 환경에 연결된 복잡한 시스템의 신뢰성을 높이기
위해 필요하다.

### Principle of Least Privilege

최소 권한 원칙은 보호의 핵심 원칙으로, 프로세스, 사용자, 시스템 모두 자신의 작업을 수행하는 데 필요한 최소한의 권한만 부여받아야 한다는 원칙이다. 불필요한 권한을 제거함으로써 악의적인 공격이나 실수로 인한
피해를 최소화할 수 있다.
대표적인 예로 UNIX의 권한 체계가 있다. root는 시스템의 최고 관리자 권한이고, sudo는 일반 사용자가 특정 명령어에 한해 관리자 권한으로 실행할 수 있도록 하는 명령어다. chmod는 파일이나 디렉토리의
접근 권한을 설정하는 명령어로, 읽기, 쓰기, 실행 권한을 사용자별로 제어할 수 있다.

### Access Matrix

보호 모델을 추상적으로 표현한 것이 접근 행렬(Access Matrix) 이다. 행렬의 행(Row)은 도메인(Domain) 을, 열(Column)은 객체(Object) 를 나타낸다. 각 항목에는 해당 도메인이 객체에
대해 가지는 접근 권한이 저장된다. 이를 ACL(Access Control List) 이라고 하며, 다양한 접근 정책을 유연하게 표현할 수 있다.

### Sandboxing

모바일에서 사용되는 정책으로 샌드박싱은 프로세스가 할 수 있는 작업을 제한된 환경에서 실행하는 방식이다. 프로세스는 자신을 실행한 사용자의 자격 증명으로 동작하며, 해당 사용자가 접근할 수 있는 자원에만 접근할 수 있다. 
Java와 .NET은 가상 머신 수준에서 샌드박싱을 적용하고, Android는 강제 접근 제어(Mandatory Access Control)의 일부로 샌드박싱을 적용한다.

### Code Signing (코드 서명)

시스템이 프로그램이나 스크립트를 신뢰할 수 있는지 어떻게 확인할 수 있을까? 코드 서명(Code Signing) 은 프로그램과 실행 파일에 디지털 서명을 적용하여 생성 이후 변조되지 않았음을 확인하는 방법이다. 운영체제 배포, 패치, 서드파티 도구뿐만 아니라 Android와 iOS의 모바일 앱에도 사용된다.
