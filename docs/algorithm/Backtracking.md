## Backtracking?

백 트래킹이란 조건을 만족하는 한 계속 검사해 나가다, 조건에 부합하지 않는 순간 탐색을 취소하고 이전 단계로 돌아온 뒤 탐색을 이어나가는 알고리즘을 말한다.  
DFS 기반의 알고리즘으로, 재귀 함수나 스택을 통해 구현 가능하다.

- 모든 가능한 경우의 수 중에서 특정한 조건을 만족하는 경우만 살펴보는 것
- DFS 등으로 모든 경우의 수를 탐색하는 과정에서, 조건문으로 컨트롤 하여 다른 경우를 탐색하게 한다. DFS와 비슷하지만, 가지치기를 통해 도중에 탐색 범위를 줄여가면서 진행하는 효율적인 방법이다.

### 코드

- 조건을 검사하기 위한 매개변수와 현재 상태를 체크하기 위한 매개변수가 필요
-

```java
void back_tracking(int count) {
	if (현재 상태가 마지막까지 도달한 경우){
		정답 카운팅;
		return;
	}

	for (가능한 모든 경우에 대해){
		// 조건을 만족하는지 검사        
		if (조건을 만족하는 경우){
			상태값 저장;
			back_tracking(count + 1);
			상태값 원복;
		}
	}
}
```

### 재귀를 써야하는 경우

#### 1. 재귀적인 자료구조나 풀이 공식이 있는 경우

**재귀적인 자료구조** : Linked List, Tree, Grid, 중첩 배열 등
**재귀적인 풀이 공식** : 팩토리얼, 피보나치 수열, 순열/조합 등

#### 2. 변수 선언 없이 코딩하고 싶을 때

재귀함 수는 호출시마다 새로운 스코프를 생성하므로 변수 없이도 문제를 풀 수 있다.

### 베이스 조건

> 답을 바로 알 수 있는 가장 간단한 상황을 가정해본다.  
> 0과 1을 사용한 인풋값을 생각해본다.

1. 베이스 조건 설계  
   베이스 조건? 더 이상 자기 자신을 호출하지 않게 하는 조건으로 재귀가 끝나는 조건을 의미한다.
   단순한 작업으로, 바로 답을 구할 수 있는 가장 쉬운 상황을 고려한다.

> 바로 답이 나오는 가장 단순한 인풋값은?

2. 베이스 조건 return  
   문제에서 요구한 답의 데이터 타입을 확인   
   문제에서 요구하는 데이터 타입 = 베이스 조건의 데이터 타입

### 분해

> 베이스 조건에 가까워지도록 인풋값을 조작한다.

#### 재귀적 분해 = 문제를 더 작게 만든다

함수를 호출할 때마다 베이스 조건에 가까워져야 한다.

### 조합

> 부분 답을 가지고 전체 답을 구하는 방법을 생각한다.

조합이란, 재귀 호출이 베이스 조건에 걸려 멈추고 나서 그 다음에 일어나는 작업

#### 1. 베이스 조건 바로 위의 단계

베이스 조건이 `n == 1` 이라면, `n = 2` 일 때  
분해는 func(n-1)

#### 2. 베이스 조건의 3단계 위 (윗 단계)

베이스 조건이 `n == 1` 이라면, `n = 4` 일 때

#### 3. 두 케이스의 공통점을 찾는다

함수가 맞다는 가정하에 `n = 2`와 `n = 4` 일 때 정확한 답이 나오는 방식을 찾는다.


> 베이스 조건 => 분해 => 조합 순서로 코드를 작성한다.
